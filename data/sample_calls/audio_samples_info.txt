## Audio Processing Notes

### For Demo Purposes:
- These files would normally contain actual Norwegian speech
- AI transcription would convert speech to text
- Speaker diarization would identify customer vs agent
- Compliance analysis would be performed on transcribed text

### Technical Requirements:
- **Supported Formats**: WAV, MP3, M4A, FLAC
- **Max File Size**: 50 MB
- **Recommended Quality**: 16kHz+ for good transcription
- **Language**: Norwegian (Bokmål/Nynorsk)

### Processing Pipeline:
1. **Upload**: Audio file uploaded via web interface
2. **Validation**: File format and size check
3. **Transcription**: Whisper AI converts speech to text
4. **Diarization**: Identify different speakers
5. **Analysis**: Norwegian compliance rule engine
6. **Redaction**: GDPR-compliant data masking
7. **Report**: Comprehensive compliance report

---

**Note**: In a production environment, these would be actual audio files. 
For this demo, the text transcriptions in .txt files represent what 
would be extracted from the audio files.
